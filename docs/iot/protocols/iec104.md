## IEC104（IEC 60870-5-104）

### ACK 机制本质

- IEC104 **没有独立 ACK 报文**
- ACK 通过以下两种方式携带：

| 类型             | 是否ACK     |
| ---------------- | ----------- |
| I帧（信息帧）    | ✅（Nr字段） |
| S帧（监督帧）    | ✅           |
| U帧（测试/控制） | ❌           |



### ACK 的定义（核心）

```
ACK = 对端发送的 Nr（Receive Sequence Number）
```

含义：

```
Nr = 对端已正确接收的 I帧数量
```



### 总召唤（C_IC_NA_1）流程

```
客户端 → I帧：ACTIVATION（总召唤）

服务端 → I帧：ACTIVATION_CON
服务端 → 多个 I帧：点表数据（带 Nr）
服务端 → I帧：ACTIVATION_TERMINATION
```

### ⚠️ 关键点

- ACTIVATION_CON ❌ 不是 ACK
- ACTIVATION_TERMINATION ❌ 不是 ACK
- **只有 I帧中的 Nr 才是 ACK**



### t1 超时（断连根因）

触发条件：

```
客户端发送 I帧后，长时间未收到对端 ACK（Nr未推进）
```

结果：

```
触发 t1 → 主动断开连接
```



### 常见误区

### 误区1：认为“有返回数据就等于ACK”

✔ 错，必须看：

```
Nr 是否推进
```

------

###  误区2：认为“总召唤响应就是ACK”

✔ 错：

```
ACTIVATION_CON / TERMINATION ≠ ACK
```

------

###  误区3：用 testCommand 当心跳

```
C_TS_NA_1 是 I帧，不是链路测试帧
```

✔ 正确心跳：

```
TESTFR_ACT / TESTFR_CON（U帧）
```



## OpenMUC 实现层经验

### ACK 处理

```java
//处理方法
handleReceiveSequenceNumber(int receiveSeqNumber)

// 核心逻辑
acknowledgedSendSequenceNumber = receiveSeqNumber;
```

### ACK 触发路径

### ✔ I帧

```java
handleIFrame()
 → handleReceiveSequenceNumber()
```

------

### ✔ S帧

```java
case S_FORMAT:
 → handleReceiveSequenceNumber()
```

### T1超时检查启动条件

```java
send(ASdu)

// 触发
timeoutManager.addTimerTask(maxTimeNoAckReceived)
```

### T1超时检查 取消条件

```java
handleReceiveSequenceNumber() {
    ...
    maxTimeNoAckReceived.cancel();
    
    // 重新启动 T1 的关键条件
    if (sendSequenceNumber != acknowledgedSendSequenceNumber) {
    	timeoutManager.addTimerTask(maxTimeNoAckReceived);
	}
}
```

### 断连直接原因

```java
MaxTimeNoAckReceivedTimer.execute()
 → close()
```

### 关键状态变量

| 变量                           | 含义              |
| ------------------------------ | ----------------- |
| sendSequenceNumber             | 已发送I帧序号     |
| acknowledgedSendSequenceNumber | 已确认I帧序号     |
| receiveSequenceNumber          | 已接收对端I帧序号 |



### S帧发送机制（客户端ACK）

触发条件：

```java
numUnconfirmedIPdusReceived >= threshold 或 maxTimeNoAckSentTimer
// 行为：
sendSFormatPdu()
```
